# Output目录文本处理器

这是一个独立的Python脚本，用于监控`output/`目录下的新章节文件，并应用文本处理规则。

## 功能特性

1. **文件监控**：自动检测`output/`目录下的新`.txt`文件
2. **文本处理规则**：
   - **规则1**：删除短引号 - 如果一组引号（单引号或双引号）包裹的是五个字以下且里面没有标点符号（比如逗号句号）的时候就把引号删除
   - **规则2**：处理"不是...，..是..."句式 - 两种处理方式：
     - 删除"不是...，而"，保留"是"和之后的内容
     - 删除"不是...，...是"，只保留"是"之后的内容，"是"也不要
   - **规则3**：删除小括号内容 - 直接删除括号和内容，比如"（好啊）"，直接删除
   - **规则4**：处理"没有...没有...只有..."句式 - 删除所有"没有"打头的句子，删除后句的"只有"，保留"只有"之后的内容
3. **版式重排功能**：
   - **分段**：每遇到句号、感叹号、问号单独分一段，引号包裹内的对话内容排除
   - **空行规范化**：消除多余空行，每段之间加一个空行，符合网文排版规范
4. **输出目录支持**：处理后的文件默认保存到`C:\Users\chucky\Nutstore\1\obsidian\MyNovel\刘斗斗`，不覆盖原文件
5. **状态持久化**：记录已处理的文件，支持断点续传
6. **多种运行模式**：单次处理、持续监控、单个文件处理

## 使用方法

### 1. 持续监控模式（默认）

默认行为：持续监控output目录，每30秒检查一次新文件：

```bash
python process_output.py
```

### 2. 单次处理模式

处理所有未处理过的新文件后退出：

```bash
python process_output.py --once
```

### 3. 处理单个文件

```bash
python process_output.py --input output/第1章.txt --output output/第1章_processed.txt
```

如果不指定`--output`参数，则会覆盖原文件。

### 4. 禁用版式重排

```bash
python process_output.py --no-reformat
```

### 5. 查看帮助

```bash
python process_output.py --help
```

## 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--once` | 单次处理模式（处理完所有新文件后退出） | `false`（持续监控） |
| `--interval` | 持续监控时的检查间隔（秒） | `30` |
| `--input` | 单个文件处理模式：输入文件路径 | - |
| `--output` | 单个文件处理模式：输出文件路径 | - |
| `--watch-dir` | 监控的目录 | `output` |
| `--output-dir` | 输出目录，处理后的文件保存到此目录 | `C:\Users\chucky\Nutstore\1\obsidian\MyNovel\刘斗斗` |
| `--no-reformat` | 禁用版式重排功能 | `false`（启用） |
| `--verbose`, `-v` | 详细输出模式 | - |

## 状态管理

脚本会在`data/text_processor_state.json`文件中保存已处理文件的状态，确保：
- 同一文件不会被重复处理
- 支持程序重启后的断点续传
- 记录处理时间戳

## 故障排除

### 问题1：编码错误
**症状**：控制台输出乱码或Unicode错误
**解决**：确保系统支持UTF-8编码，或使用`--verbose`参数查看详细日志

### 问题2：文件未被处理
**症状**：文件存在但脚本报告"没有新文件需要处理"
**解决**：检查状态文件`data/text_processor_state.json`，删除对应条目或整个文件重置状态

### 问题3：处理效果不符合预期
**症状**：文本处理规则没有正确应用
**解决**：使用测试脚本验证规则，或调整`text_processor.py`中的正则表达式

## 文本处理规则详解

### 规则4：处理"没有...没有...只有..."句式
**功能**：删除所有"没有"打头的句子，删除后句的"只有"，保留"只有"之后的内容

**示例**：
- **处理前**："没有白光，没有眩晕，更没有引导精灵。只有眼前一黑，像熬夜熬狠了突然断片。"
- **处理后**："眼前一黑，像熬夜熬狠了突然断片。"

**支持的变体**：
1. "没有A，没有B，只有C。"
2. "没有A，没有B，更没有C。只有D。"
3. "没有A，没有B。只有C。"
4. "没有A、没有B、只有C。"

### 规则5：删除Markdown强调语法
**功能**：删除Markdown强调标记，但保留内容

**支持的标记**：
1. **内容** - 粗体（双星号）
2. *内容* - 斜体（单星号）
3. __内容__ - 粗体（双下划线）
4. _内容_ - 斜体（单下划线）

**示例**：
- **处理前**："**清晰的、五指张开的血手印**"
- **处理后**："清晰的、五指张开的血手印"

**嵌套处理**：
- **处理前**："**粗体中的*斜体***"
- **处理后**："粗体中的斜体"

**混合内容**：
- **处理前**："**重要提示**：这是一个*测试*内容，包含__多个__标记。"
- **处理后**："重要提示：这是一个测试内容，包含多个标记。"

## 版式重排功能详解

### 分段规则
1. **句子分段**：每遇到句号(。)、感叹号(！)、问号(？)单独分一段
2. **对话处理**：引号包裹内的内容作为整体处理，不按标点分段
3. **引号内不分段**：引号内的内容（包括中文方括号【】、书名号《》等）遇到换行符也不分段
4. **支持的引号类型**：单引号(')、双引号(")、中文引号(「」『』)、中文方括号(【】)、书名号(《》)、尖括号(〈〉)、花括号(〔〕)
5. **混合对话**：如果对话分成了两部分（如："不知道。" 刘斗斗老实回答，"从老家带来的"），前面那部分对话分段，后面的另分一段

### 空行规范化
1. **消除多余空行**：先消除分段之间的所有空行，使格式统一
2. **添加标准空行**：然后每段之间加一个空行，符合网文排版规范

### 引号处理改进
**问题修复**：解决了中文方括号【】内的内容被错误分段的问题

**示例**：
- **处理前**：
  ```
  测试中文方括号内的分段问题。
  
  【想明白生命的意义吗？
  
  想真正地……活着吗？
  
  】
  ```
- **处理后**：
  ```
  测试中文方括号内的分段问题。
  
  【想明白生命的意义吗？想真正地……活着吗？】
  ```

### 示例
**处理前**：
```
刘斗斗觉得自己的肺像两个破风箱。每一次吸气都带着巷子里那股铁锈味。"不知道。"刘斗斗老实回答，"从老家带来的，挺结实。"
```

## 问题修复记录

### 2026-01-13：修复三个关键问题

根据用户反馈，修复了以下三个问题：

#### 1. 短引号删除规则优化
**问题**：短引号删除规则没有正确工作
**修复**：改进了`process_short_quotes()`方法，现在能正确识别各种引号类型（单引号、双引号、中文引号等），并准确判断5字以下且无标点的引号内容
**规则**：
- 引号包裹的内容在5个字以下（中文字符计数）
- 内容中没有标点符号（逗号、句号、感叹号、问号、分号、顿号）
- 符合条件时删除引号，保留内容
- 不符合条件时保留原样

**示例**：
- `"好"` → `好`（单字，无标点）
- `"不好"` → `不好`（两字，无标点）
- `"别开，"` → `"别开，"`（有逗号，保留引号）
- `"噗"声` → `噗声`（单字，无标点）

#### 2. 对话内容被错误分段问题
**问题**：完整的对话内容被切割成了好几段
**修复**：改进了`LayoutFormatter.split_into_paragraphs()`方法，确保完整的对话内容不被分段
**规则**：
- 引号内的内容（对话）作为一个整体处理
- 即使引号内有句号、感叹号、问号，也不分段
- 引号外的句子结束标点正常分段

**示例**：
- `"别开，"他说，"可能是陷阱。"` → 保持为一个段落
- `外面下雨了。"真糟糕。"他心想。` → 分为两个段落（`外面下雨了。`和`"真糟糕。"他心想。`）

#### 3. 段落首行缩进问题
**问题**：有些段落前面有空格，应该顶格（无首行缩进）
**修复**：在`TextProcessor.process_text()`方法中添加了去除段落前空格的逻辑
**规则**：
- 处理完成后，删除每行开头的空格和制表符
- 保留空行不变
- 确保所有段落顶格显示，符合网文排版规范

**示例**：
- `    前面有四个空格。` → `前面有四个空格。`
- `\t前面有制表符。` → `前面有制表符。`
