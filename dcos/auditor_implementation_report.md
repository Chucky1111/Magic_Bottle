# Auditor模块实现审计报告

## 项目概述
本项目为AI自动小说创作系统实现了auditor模块，用于在每章生成后进行质量审计。系统采用"高内聚、低耦合"的模块化架构，严格遵循项目宪法要求。

## 技术架构详解

### 1. 3-2-1审计逻辑实现

```python
# 审计流程
1. 审计（audit） - 3次尝试
2. 重写（rewrite） - 2次尝试  
3. 确认（confirm） - 1次最终确认
```

### 2. 模块化架构

```
config/              # 配置管理
├── settings.py      # 系统配置
core/                # 核心底层
├── llm.py           # LLM客户端封装
├── auditor_parser.py # 审计解析器
storage/             # 数据存取
├── __init__.py      # JSON读写
services/            # 业务逻辑
├── workflow.py      # 主写作流程
├── auditor_workflow.py # 审计工作流
├── auditor_state_manager.py # 审计状态
└── pruning_strategy.py # 剪枝策略
```

### 3. 状态持久化机制

- **写作状态**：`data/state.json`
- **审计状态**：`data/auditor_state.json`
- **历史记录**：`data/history.json`
- **断点续传**：每一步操作后立即写入硬盘

### 4. 健壮性保障

- **重试机制**：使用`tenacity`库实现工业级重试
- **异常处理**：网络超时、API限流、JSON解析错误
- **超时控制**：所有API调用都有超时设置
- **日志记录**：详细的运行日志和错误追踪

## 关键创新点

### 1. 中间变量审计机制
- 写作生成内容→审计→通过写入output，不通过重写
- 避免直接污染输出目录
- 保持创作流程的纯净性

### 2. 历史记录动态更新
- 重写后自动更新历史记录
- 保持上下文一致性
- 避免历史记录与最终输出不一致

### 3. 提示词模板化
- 所有提示词存储在`prompts/Auditor/`目录
- 支持变量替换和模板引擎
- 便于维护和调整

### 4. 配置驱动
- 所有参数通过配置文件管理
- 无需修改代码即可调整审计标准
- 支持环境变量和配置文件
