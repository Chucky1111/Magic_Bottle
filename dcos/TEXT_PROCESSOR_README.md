# 文本处理器 (Text Processor) - 完整解决方案

## 概述

这是一个监控 `output/` 目录的脚本，当有新章节文件时自动激活，应用多个文本处理规则，并进行版式重排。

## 核心功能

### 1. 文件监控
- 监控 `output/` 目录下的新 `.txt` 文件
- 支持断点续传（状态保存在 `data/processor_state.json`）
- 自动处理新文件，避免重复处理

### 2. 文本处理规则

#### 规则1：短引号删除
- 删除五个字以下且没有标点符号的引号内容
- 支持中文引号 `"` 和 `"`
- 示例：`"力量"` → `力量`

#### 规则2："不是...，...是..." 句式处理
- 两种处理方式：
  1. 删除 `"不是...，而"`，保留 `"是"` 和之后的内容
  2. 删除 `"不是...，...是"`，只保留 `"是"` 之后的内容（不包括 `"是"`）

#### 规则3：括号内容删除
- 删除所有小括号及其内容
- 示例：`"（好啊）"` → `""`

#### 规则4："没有...只有..." 句式处理
- 删除 `"没有...只有"`，保留 `"只有"` 之后的内容

#### 规则5：Markdown强调语法删除
- 删除 `**` 和 `*` 强调语法
- 示例：`"**重要**"` → `"重要"`

### 3. 版式重排 (Layout Formatter)

#### 对话分段修复
- 引号内的换行符不分段
- 单独成行的引号与前一行合并
- 支持多种引号类型：
  - 英文引号：`"`, `'`
  - 中文引号：`「`, `」`, `『`, `』`, `"`, `"`, `"`, `"`
  - 中文括号：`【`, `】`, `〔`, `〕`
  - 中文书名号：`《`, `》`, `〈`, `〉`

#### 段落规范化
- 删除多余空行
- 确保段落间只有一个空行
- 修复首行缩进问题

## 修复历史

### 2026-01-13 修复的问题

#### 1. 短引号删除问题
- **问题**：中文引号 `"力量"` 和 `"回应"` 没有被删除
- **原因**：正则表达式使用 `"` 匹配，但中文引号是 Unicode 字符 `"` (U+201C) 和 `"` (U+201D)
- **修复**：使用 Unicode 转义序列 `\u201c` 和 `\u201d` 匹配中文引号

#### 2. 对话分段问题
- **问题**：对话内容被错误分段，引号 `"` 被单独放在一行
- **原因**：
  - `split_into_paragraphs` 方法正确识别引号，但 `normalize_paragraphs` 中的引号字符集不完整
  - `_is_quote_only_line` 方法缺少中文引号 `"` 和 `"`
- **修复**：
  - 扩展引号字符集，添加 `'“'` 和 `'”'`
  - 确保 `normalize_paragraphs` 能正确识别单独成行的中文引号

#### 3. 引号配对支持
- **增强**：添加对多种中文引号类型的支持
- **实现**：扩展 `quote_pairs` 字典，支持 `"`/`"` 配对
