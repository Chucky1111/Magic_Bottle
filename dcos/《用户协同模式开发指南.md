# 用户协同模式开发指南

## 1. 问题分析

### 1.1 当前行为：变量为空时的情况

当前实现中，当用户灵感变量为空时（即没有可用的灵感条目），系统会：

1. **继续运行，不会暂停**：在 `render_design_prompt()` 和 `render_write_prompt()` 方法中，如果 `get_next_idea()` 返回 `None`，代码会记录调试日志但不会中断流程。
2. **优雅降级**：用户灵感内容会被设置为空字符串，模板中的 `{{user_idea}}` 占位符会被替换为空值。
3. **不影响核心功能**：小说写作流程会正常进行，只是缺少用户灵感注入。

相关代码片段：
```python
# services/prompt_renderer.py 第217-220行
else:
    logger.debug(f"第{chapter_num}章没有可用的用户灵感")
```

**结论**：当前实现符合"健壮性优先"原则，变量为空时系统继续运行。

### 1.2 用户需求：协同模式

用户希望实现一个"用户协同模式"，具体功能包括：

1. **统计能力**：能够检索有多少个用户灵感（已实现 `get_stats()` 方法）
2. **自动章节规划**：根据灵感数量自动设置 `--cc` 参数（额外运行的章节数）
3. **配置开关**：通过 `.env` 文件开启/关闭此模式
4. **向后兼容**：未开启时，用户灵感为空也能正常运行

### 2.2 统计信息增强

扩展 `UserIdeaManager.get_stats()` 方法，提供更多协同模式所需信息：

```python
def get_collaborative_stats(self) -> Dict[str, Any]:
    """获取协同模式统计信息"""
    base_stats = self.get_stats()
    
    # 计算建议的章节数
    pending_entries = base_stats["pending_entries"]
    min_chapters = self.config.get("min_chapters_per_idea", 1)
    suggested_chapters = pending_entries * min_chapters
    
    return {
        **base_stats,
        "suggested_chapters": suggested_chapters,
        "has_enough_ideas": pending_entries >= 1,
        "collaborative_mode_enabled": self.config.get("collaborative_mode", False)
    }
```

## 5. 配置与使用说明

### 5.1 启用协同模式

在 `.env` 文件中设置：

```env
USER_IDEA__COLLABORATIVE_MODE=true
USER_IDEA__AUTO_CC_ENABLED=true
USER_IDEA__MIN_CHAPTERS_PER_IDEA=2  # 每个灵感分配2章
```

### 5.2 命令行使用示例

```bash
# 基本使用：自动根据灵感数量决定运行章节数
python main.py

# 协同模式激活时，如果有5个待处理灵感，min_chapters_per_idea=2
# 则会自动运行 5 × 2 = 10 章

# 手动覆盖：即使协同模式开启，也能手动指定
python main.py --cc 5  # 优先使用手动指定的5章

# 目标章节优先：--ct 优先级最高
python main.py --ct 50  # 运行到第50章，忽略协同模式
```

## 6. 异常处理与降级策略

### 6.1 统计失败

如果获取灵感统计失败，系统将：

1. 记录警告日志
2. 回退到默认停止逻辑
3. 继续运行，不会中断

### 6.2 配置冲突

处理配置冲突的优先级：

1. `--ct` (目标章节) - 最高优先级
2. `--cc` (额外章节) - 中优先级  
3. 协同模式自动计算 - 低优先级
4. 默认逻辑（硬编码30章） - 最低优先级

### 6.3 灵感耗尽处理

如果在运行过程中灵感耗尽：

1. 已完成章节会正常保存
2. 后续章节将没有用户灵感注入
3. 系统会记录警告并继续运行
4. 不会因为灵感耗尽而提前停止

### 7.2 智能分配算法

```python
def calculate_suggested_chapters(self, pending_ideas: List[IdeaEntry]) -> int:
    """智能计算建议章节数"""
    # 基于灵感长度、复杂度、优先级等因素
    total_complexity = sum(idea.calculate_complexity() for idea in pending_ideas)
    base_chapters = max(1, total_complexity // 100)  # 每100复杂度分配1章
    return min(base_chapters, 50)  # 上限50章
```
