# 玻璃球游戏：内在混沌的显现

## 设计思路

本游戏源于对"表象与内核"的深度探索。玻璃球象征人类对纯净、透明、完美的追求，而其内部暗流涌动的混沌则揭示万物本质的不可净化性。游戏试图让玩家体验：在光滑表面之下，存在着永恒运转的浑浊核心——那正是维持存在的原始动力。

## 风格选择

### 视觉风格
- **色调**：以深黑、暗红、褐灰为主，避免鲜艳色彩。色彩饱和度低，明度低，营造压抑而深邃的氛围。
- **纹理**：使用细微的裂纹、污渍、磨损效果，在UI元素和游戏物件上暗示内在的腐朽。
- **光照**：破碎的光线，不均匀的反射，刻意制造视觉上的"不完美"感。
- **动画**：流体运动采用湍流、漩涡等不规则模式，避免机械的匀速运动。

### 交互风格
- **操作反馈**：延迟、阻力、粘滞感，模拟在泥泞中挣扎的体验。
- **声音设计**：低频噪音、破裂声、液体流动声，避免清脆悦耳的音效。
- **界面布局**：不对称、重叠、边缘模糊，打破传统的整洁排列。

### 游戏机制
- **核心玩法**：玩家操控玻璃球在有限空间中移动，球内流体随操作而变化。
- **目标**：没有明确的"胜利"，而是让玩家在探索中逐渐接受混沌的不可驾驭性。
- **进程**：随着时间推移，球体表面出现更多裂纹，内部流体逐渐"污染"外部空间。

## 技术实现
- 使用TypeScript编写逻辑，确保类型安全与可维护性。
- 基于Canvas 2D渲染，实现动态流体模拟。
- 采用Bun作为包管理与开发环境，提升开发效率。
- 纯前端实现，无需后端，单HTML文件即可运行。

## 哲学立场
本游戏不追求愉悦的娱乐体验，而是试图呈现一种存在主义的沉思：我们竭力维持的秩序与洁净，其下永远是涌动的不洁与混沌。接受这种混沌，不是放弃，而是对真实更深层的拥抱。

## 运行方式
1. 安装依赖：`bun install`
2. 启动开发服务器：`bun run dev`
3. 在浏览器中打开 `http://localhost:3000`

## 文件结构
- `index.html` 主页面
- `src/main.ts` 游戏入口
- `src/fluid.ts` 流体模拟核心
- `src/glassball.ts` 玻璃球渲染与交互
- `styles/` CSS样式
- `assets/` 图像与音效资源
