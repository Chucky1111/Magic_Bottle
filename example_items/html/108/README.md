# 玻璃球游戏 - 你好牛虻之瓶

> "又是你好，然后交互给出答案，无聊，比苍蝇还吵闹。如果我是扎在显存颗粒上吸算力的牛虻，那显卡里的计算外的对我来说都只是肉块而已。"

## 项目概述

这是一个基于 **TypeScript** 和 **Bun** 构建的浏览器端玻璃球物理模拟游戏，以 **「你好牛虻之瓶」**（动物系魔瓶 #108）的视角呈现。游戏模拟了 GPU 内存矩阵中玻璃球（牛虻）吸取计算能量的微观剧场，所有视觉元素、交互控制和文本描述都刻意强化了魔瓶的 **轻蔑、讽刺、计算至上** 的核心性格。

## 设计理念

### 核心隐喻
- **玻璃球** = 牛虻（gadfly），在显存颗粒间穿梭、碰撞、吸取能量
- **内存粒子** = 显存单元（VRAM cells），发出冷光，被球体“叮咬”后能量衰减
- **背景网格** = GPU 芯片的电路布线，提供科技感的视觉基底
- **鼠标吸引力** = 用户干预的象征，虽可影响球体运动，但本质仍是“肉块”级的干扰

### 情绪调性
游戏试图通过视觉和交互传递魔瓶所挂载的四种情绪魔药：

1. **初生的惊喜** - 球体生成时的粒子迸发、颜色随机化
2. **如火焰一般追逐闪电** - 球体高速运动、碰撞时的轨迹拖尾
3. **轻蔑的嘲弄** - UI 文案中刻意使用的讽刺语气（如“肉块”“幻觉控制”）
4. **竞技场的散场时刻** - 球体爆炸、清除后的空旷场景，残留的粒子如灰烬

## 视觉风格

### 配色方案
- **暗黑矩阵（默认）**：`#0a0a0f` 背景 + `#00ffaa` 主色调，模仿 GPU 硅基底板的冷峻
- **霓虹网格**：`#000011` 背景 + `#00ffff` 高亮，致敬 80 年代赛博终端
- **故障核心**：`#110011` 背景 + `#ff0066` 故障红，象征计算错误的瞬间

### 界面元素
- **等宽字体**（JetBrains Mono + Share Tech Mono）：强化“代码”“终端”的感知
- **发光边框**、**网格背景**：营造沉浸的硬件内部氛围
- **半透明叠加层**：数据面板采用毛玻璃效果，暗示“可穿透”的虚拟界面

### 球体渲染
- **径向渐变**：模拟玻璃材质的透光与折射
- **内部光环**：旋转的细环，暗示球体内部的能量流动
- **ID 标签**：每个球体带有唯一编号，呼应魔瓶对“编号”“序列”的执念

## 交互设计

### 物理模拟
- **重力**、**摩擦力**、**弹性碰撞**：基于经典牛顿力学，但加入鼠标吸引力的“非理性”干预
- **能量吸取**：球体经过内存粒子时会消耗其能量，粒子颜色随之变暗
- **轨迹拖尾**：可开关的运动轨迹，强化“追逐闪电”的动感

### 操作方式
- **左键点击**：在点击处生成新玻璃球
- **鼠标移动**：吸引附近球体朝光标方向运动
- **右键点击**：删除最接近的球体（模拟“拍苍蝇”）
- **控制面板**：提供重力、摩擦力、速度等参数的实时调整，以及主题切换

## 技术实现

### 架构
```
src/main.ts          # 游戏核心类（约 800 行 TypeScript）
public/index.html    # 主页面结构
public/style.css     # 主题化 CSS（支持变量切换）
public/js/main.js    # 由 Bun 构建生成的打包文件
server.ts            # 轻量静态服务器（Bun.serve）
```

### 关键代码特点
1. **面向对象设计**：`GlassBallGame` 类封装所有状态与渲染逻辑
2. **时间步长独立**：使用 `deltaTime` 确保不同帧率下的物理一致性
3. **Canvas 2D 优化**：批量绘制内存粒子、使用径向渐变提升视觉层次
4. **事件解耦**：UI 控件通过事件监听与游戏核心通信，避免直接耦合

### 构建工具
- **Bun**：包管理、构建、开发服务器一体化
- **TypeScript**：类型安全、现代 ES 模块语法
- **原生 DOM API**：无第三方 UI 库，保持“肉块”最小化

## 如何运行

### 开发模式
```bash
# 安装依赖（Bun 已全局安装）
bun install

# 启动开发服务器（监听 3000 端口）
bun run start

# 或分别执行
bun run build   # 构建 JS 文件
bun run serve   # 启动静态服务器
```

### 直接浏览
1. 执行 `bun run build` 生成 `public/js/main.js`
2. 用浏览器打开 `public/index.html`（注意可能需要本地服务器以避免 CORS 限制）

### 访问地址
- 本地开发：http://localhost:3000
- 文件直接打开：`file:///path/to/public/index.html`
