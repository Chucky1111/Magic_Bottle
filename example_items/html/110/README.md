# 玻璃球游戏

一个基于Canvas的物理模拟游戏，展示玻璃态美学与物理交互。

## 设计思路

### 核心理念
玻璃球游戏的核心设计理念是"有序中的混沌"。游戏模拟了玻璃球在物理环境中的运动、碰撞与互动，每个玻璃球都遵循简单的物理规律，但众多球体相互作用时会产生复杂且不可预测的轨迹变化，体现了从简单规则到复杂涌现的哲学思考。

### 视觉设计
1. **玻璃态美学**：界面采用半透明材质、柔和阴影和渐变色彩，模仿真实玻璃的光学特性
2. **动态光影**：玻璃球表面具有实时高光效果，根据环境和运动状态变化
3. **色彩系统**：主色调为蓝绿渐变，灵感来自清晨露珠和水晶棱镜的折射效果
4. **毛玻璃效果**：控制面板采用背景模糊效果，增强层次感和深度

### 交互设计
1. **直观操作**：点击添加、拖拽投掷、滑动调节，所有操作符合直觉
2. **实时反馈**：玻璃球碰撞时高亮显示，拖拽时有轨迹线提示
3. **参数调节**：重力开关、摩擦力滑块，让用户能探索不同物理参数下的行为变化

## 技术实现

### 架构
- **前端**：纯HTML/CSS/TypeScript，无任何框架依赖
- **物理引擎**：自定义实现的2D物理模拟系统
- **渲染**：Canvas 2D API，利用径向渐变模拟玻璃材质
- **构建工具**：Bun + TypeScript编译器

### 核心技术
1. **物理模拟**：
   - 牛顿运动定律（速度、加速度）
   - 弹性碰撞检测与响应
   - 边界反弹与能量衰减
   - 重力与摩擦力模拟

2. **视觉效果**：
   - Canvas径向渐变创建玻璃质感
   - 实时高光位置计算
   - 拖拽轨迹可视化
   - 碰撞高亮动画

3. **交互系统**：
   - 鼠标与触摸事件统一处理
   - 拖拽投掷的动量计算
   - 实时UI状态更新

### 性能优化
- 使用`requestAnimationFrame`实现平滑动画
- 碰撞检测采用空间分割优化（待实现）
- Canvas渲染批次优化
- 避免内存泄漏的事件管理

## 风格选择解析

### 1. 玻璃态设计语言
```
核心理念：透明但不简单，折射而不失真
色彩系统：hsla(180-270, 60-90%, 50-70%, 0.7-1.0)
材质特性：半透明、高反射、内折射
```

### 2. 色彩心理学
- **主蓝绿色调**：象征清澈、冷静、科技感
- **渐变过渡**：营造深度和立体感
- **高饱和度**：增强视觉吸引力而不刺眼

### 3. 动效设计原则
- **缓动函数**：所有过渡使用ease-in-out
- **物理一致性**：界面动画也遵循物理规律
- **反馈及时性**：用户操作在100ms内得到视觉反馈

### 4. 字体与排版
- **标题字体**：Playfair Display - 优雅衬线字体，增强艺术感
- **正文字体**：Inter - 现代无衬线字体，保证可读性
- **等宽字体**：代码和数值显示，强调精确性

## 设计哲学思考

### 从混沌到有序
玻璃球游戏看似简单的物理模拟，实际上体现了复杂系统的几个关键特征：

1. **初始敏感性**：微小的初始速度差异会导致完全不同的轨迹
2. **非线性互动**：球体间的碰撞会产生连锁反应
3. **涌现行为**：个体遵循简单规则，群体表现出复杂模式

### 技术与艺术的平衡
项目尝试在以下维度寻求平衡：
- **计算精确性** vs **视觉表现力**
- **物理真实性** vs **游戏趣味性**  
- **代码简洁性** vs **功能完整性**
- **性能效率** vs **视觉效果**

### 玻璃的隐喻
玻璃作为一种材料，具有多重象征意义：
- **透明性**：系统状态完全可见，没有黑箱
- **脆弱性**：精细平衡容易打破
- **折射性**：同一现象在不同角度呈现不同面貌
- **记忆性**：玻璃制品保留制造过程的痕迹
