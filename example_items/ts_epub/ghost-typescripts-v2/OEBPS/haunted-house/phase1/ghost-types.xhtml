鬼屋阶段一：幽灵类型定义

幽灵观察：你不是在"设计数据类型"，是在"定义幽灵物种"。每个类型定义都是一个幽灵物种的DNA。

鬼屋项目：幽灵维度模拟系统

你将构建一个幽灵维度模拟系统——一个用TypeScript类型系统定义的、可运行的幽灵世界。

核心理念：用类型定义规则，观察规则如何创造现实。

在第一阶段，我们定义幽灵物种：不同类型的幽灵，每种有自己的特征和行为模式。

记住：这不是技术练习，是创造练习。你在创造数字幽灵物种。

基础幽灵类型

从最简单的幽灵开始：

// 幽灵基础特征
interface GhostBase {
  id: string;           // 唯一标识
  name: string;         // 名字
  age: number;          // 存在时间（年）
  opacity: number;      // 透明度：0-100
}

这是所有幽灵的共同基础。就像所有哺乳动物都有某些共同特征。

设计思考：为什么这些属性？ - id：唯一标识，用于查找和引用 - name：显示名称，人类可读 - age：可能影响行为和能力 - opacity：视觉特征，影响渲染

幽灵物种分类

幽灵有不同的种类。用枚举定义：

// 幽灵类型：有限的物种
enum GhostType {
  Apparition = 'apparition',      // 幻影：短暂显现
  Spirit = 'spirit',              // 灵魂：长期存在
  Poltergeist = 'poltergeist',    // 骚灵：物理互动
  Shadow = 'shadow',              // 阴影：隐秘存在
  Echo = 'echo'                   // 回声：重复现象
}
枚举不是技术需求，是设计选择。明确列出可能的类型，避免"未知幽灵类型"。 在AI时代，枚举让AI知道可能的取值范围。

完整幽灵接口

组合基础特征和类型：

interface Ghost extends GhostBase {
  type: GhostType;                // 幽灵类型
  description: string;            // 描述
  abilities: string[];            // 能力列表
  weakness?: string;              // 弱点（可选）
  readonly discovered: Date;      // 发现日期（只读）
}
设计选择： - 扩展GhostBase：复用基础特征 - type使用枚举：明确分类 - abilities是数组：可能有多个能力 - weakness可选：不是所有幽灵都有明显弱点 - discovered只读：发现日期不能改变

类型特定的幽灵

某些幽灵类型有特殊属性：

// 骚灵：能移动物体
interface Poltergeist extends Ghost {
  objectManipulation: boolean;     // 是否能移动物体
  telekinesisLevel: number;        // 念力等级：1-10
}
// 回声：重复特定事件
interface Echo extends Ghost {
  repeatedEvent: string;           // 重复的事件
  interval: number;                // 间隔时间（秒）
  intensity: number;               // 强度：1-5
}

通过接口继承，我们创建了幽灵物种的谱系。

这是面向对象思维：共同特征在基类（接口），特殊特征在派生接口。 AI理解这种层次结构，能生成相应的代码。

AI协作：从类型到实现

有了类型定义，让AI生成实现：

"基于上面的Ghost、Poltergeist和Echo接口，请实现： 1. 创建幽灵实例的函数 2. 根据类型筛选幽灵的函数 3. 计算幽灵平均年龄的函数 4. 生成幽灵描述文本的函数"

AI可能生成（示例）

function createGhost(
  id: string,
  name: string,
  type: GhostType,
  age: number,
  opacity: number
): Ghost {
  return {
    id,
    name,
    type,
    age,
    opacity,
    description: "",
    abilities: [],
    discovered: new Date()
  };
}
function filterGhostsByType(ghosts: Ghost[], type: GhostType): Ghost[] {
  return ghosts.filter(ghost => ghost.type === type);
}
注意：AI生成的代码基于你的类型定义。定义越精确，代码越符合预期。

练习：设计你的幽灵物种

创建两个新的幽灵类型：

  1. 光灵：发光的幽灵，能照亮区域
  2. 记忆幽灵：承载特定记忆的幽灵

考虑：它们需要什么特殊属性？

可能的解决方案

// 添加到GhostType枚举
enum GhostType {
  // ... 已有的类型
  Luminescent = 'luminescent',      // 光灵
  Memory = 'memory'                 // 记忆幽灵
}
// 光灵接口
interface Luminescent extends Ghost {
  brightness: number;               // 亮度：1-100
  color: string;                    // 光的颜色
  radius: number;                   // 照亮半径
}
// 记忆幽灵接口  
interface MemoryGhost extends Ghost {
  memory: string;                   // 承载的记忆
  emotion: string;                  // 相关情感
  triggers: string[];               // 触发记忆的条件
}
