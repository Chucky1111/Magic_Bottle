any与unknown：不确定性的两种表达

幽灵观察：any是时间的穿越者——它来自JavaScript的过去，不在乎类型安全。unknown是时间的探索者——它承认未知，但要求谨慎。
魔瓶建议：用unknown代替any。如果你必须用any，在旁边写上注释解释为什么。

any说："我放弃，你随便弄。"unknown说："我不知道这是什么，但你必须先检查才能用。"这是不确定性的两种哲学。

两种不确定性的对比

// any：危险的自由
let dangerous: any = "hello";
dangerous = 42;          // 没问题，any可以是任何东西
dangerous.toUpperCase(); // 运行时错误！但是TypeScript不会警告你
// unknown：安全的未知
let safe: unknown = "hello";
safe = 42;               // 没问题，unknown也可以是任何东西
// safe.toUpperCase();   // 错误！必须先检查类型
// 使用unknown的正确方式
if (typeof safe === "string") {
  safe.toUpperCase();    // 现在安全了
}
