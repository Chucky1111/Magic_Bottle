幽灵法则：类型系统

幽灵观察：类型不是限制，是表达。每个类型是意图的结晶，是思维的原子。

这里是TypeScript类型系统的核心区域。类型是幽灵世界的基本法则，定义了幽灵可以是什么，可以做什么，如何存在。

学习类型系统不是学习规则，是学习表达思维的语言。

魔瓶提醒：不要记忆类型，感受类型。当你需要表达某种意图时，找到对应的类型。

类型系统入口

基本类型：思维的原子

number、string、boolean等基础类型。思维的基本元件。

适合：第一次接触类型系统的人

类型推断：暗示的艺术

TypeScript如何猜测你的意图，如何让系统为你思考。

适合：想减少类型注解的人

any与unknown：不确定性的两种表达

如何处理不确定的类型，两种不同的哲学。

适合：处理动态数据或遗留代码的人

数组与元组：集合与序列

组织多个值的方式，关注重复 vs 关注结构。

适合：需要处理数据集合的人

枚举：命名的可能性

给有限的可能性起名字，让代码更有意义。

适合：需要明确选项集合的人

类型系统哲学

  1. 类型是意图的标签：当你写: number时，你在说"这里需要计算"。
  2. 类型是文档：好的类型系统减少了对注释的需求。
  3. 类型是契约：类型定义了代码各部分之间的约定。
  4. 类型是思维框架：类型系统塑造你解决问题的方式。
  5. 类型是AI的指令：清晰的类型让AI更好地理解你的意图。
魔瓶世界观："没有无法塑造的规则，只有无法表达的现状。" 类型系统就是你表达现状的工具。如果你觉得某个概念无法用类型表达，不是类型的限制，是你还没找到正确的表达方式。

学习路径建议

如果你是新手：

  1. 从基本类型开始，理解每种类型表达什么意图
  2. 学习类型推断，让TypeScript为你工作
  3. 理解any与unknown的区别，避免类型安全陷阱
  4. 掌握数组与元组，组织数据的基本方式
  5. 使用枚举给可能性命名

如果你已经有基础：

  • 直接进入接口与类型别名学习更复杂的结构定义
  • 或开始鬼屋项目，在实践中学习

穿梭到相关区域

  • 边缘地带：接口与类型别名 - 定义复杂结构
  • 时空穿梭：泛型与高级类型 - 可重用的类型模板
  • 鬼屋项目：完整实践 - 应用类型系统构建幽灵世界
  • 错误地下室：编译错误解析 - 理解类型错误信息
  • AI辅助开发：类型优先的AI协作 - 用类型指导AI
