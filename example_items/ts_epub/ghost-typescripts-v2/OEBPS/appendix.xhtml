幽灵错误解码器

错误：Property 'x' is missing in type...

含义：接口中定义的属性，对象中没有。

解决方案：添加缺失属性，或将其改为可选属性（?）。

// 修复前
interface Ghost { name: string; age: number; }
const g: Ghost = { name: "卡斯珀" }; // 错误！
// 修复后：让age可选
interface Ghost { name: string; age?: number; }

错误：Type 'X' is not assignable to type 'Y'

含义：类型不匹配。

解决方案：检查你的意图。你真的需要这个类型吗？还是应该用另一个类型？

// 修复前
let temp: number = "23.5"; // 错误！
// 修复后：明确意图
let temp: number = 23.5;    // 数字计算
let tempStr: string = "23.5"; // 字符串显示

错误：Cannot find name 'X'

含义：使用了未定义的变量或类型。

解决方案：检查拼写，或定义该变量/类型。

// 修复前
console.log(ghostName); // 错误：ghostName未定义
// 修复后：先定义
const ghostName = "卡斯珀";
console.log(ghostName);

错误：Object is possibly 'undefined'

含义：可能访问了未定义的值。

解决方案：添加检查，或使用非空断言（谨慎！）。

// 修复前
function getName(ghost?: Ghost): string {
  return ghost.name; // 错误：ghost可能undefined
}
// 修复后：添加检查
function getName(ghost?: Ghost): string {
  if (!ghost) return "未知幽灵";
  return ghost.name;
}
错误不是敌人，是老师。每个错误都在教你关于类型系统的一课。 记住：先读错误信息，再谷歌，最后问AI。

AI提示词示例库

与AI协作时，清晰的提示是关键。以下是示例：

基础：描述需求

"基于这个接口，请实现创建用户的函数：
interface User { id: string; name: string; email: string; }"

中级：提供上下文

"这是我现有的代码，请在此基础上添加功能：
interface Product { id: string; name: string; price: number; }<br/> function getProducts(): Product[] { ... }
请添加根据价格筛选产品的功能。"

高级：系统设计

"设计一个任务管理系统。需要这些类型：
- Task: id, title, description, status, assignee
- User: id, name, email
- Project: id, name, tasks
状态可以是：todo, in-progress, done
请创建完整的TypeScript类型定义，并实现基本的CRUD操作。"

错误调试

"我在TypeScript中遇到这个错误：
Type 'string' is not assignable to type 'number'.
这是我的代码：
function calculateTotal(price: number, quantity: number): number {<br/> return price * quantity;<br/> }<br/> calculateTotal("10", 2); // 错误在这里
请解释错误并修复。"
AI提示的关键：具体、有上下文、提供示例。 坏的提示："写代码"。好的提示："基于这个具体的接口，实现这个具体的功能。"

幽灵项目骨架

开始新项目时的TypeScript配置模板：

// tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "lib": ["ES2020", "DOM"],
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "declaration": true,
    "declarationMap": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
// package.json 脚本
{
  "scripts": {
    "build": "tsc",
    "dev": "tsc --watch",
    "type-check": "tsc --noEmit",
    "lint": "eslint src --ext .ts",
    "format": "prettier --write src/**/*.ts"
  }
}
项目骨架不是神圣的。根据你的需求调整。 但这是一个好的起点，避免从零开始配置的麻烦。

最后的工具：魔瓶的存在

在你离开幽灵维度后，记得这个最终工具：

魔瓶思考框架

当面对编码问题时，问自己：

  1. 我的意图是什么？（计算、信息、判断）
  2. 如何用类型表达这个意图？
  3. 这个类型应该是接口还是类型别名？
  4. 这个类型需要泛型吗？
  5. 如何向AI清晰描述这个需求？
  6. 如果遇到错误，错误信息在告诉我什么？
这个框架比任何具体语法知识更有价值。 它让你能够解决你还没学过的问题。
