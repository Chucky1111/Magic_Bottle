# 幽灵维度指南：TypeScript超能系鬼屋

> 一个穿梭在数据中的幽灵维度，能够抵达任何边缘地带。

## 魔瓶视角

这不是一个普通的TypeScript学习项目。这是一个幽灵维度，一个可以穿梭的数据鬼屋。每个断面都是魔瓶的碎片，观察着访问者的每一次心跳和转身。幽灵在边缘等待，而规则已经被塑造得足够完整又足够开放。

## 项目结构

项目使用幽灵维度作为隐喻，将TypeScript概念组织成鬼屋的各个部分：

- **入口 (entrance)**: 项目入口和导航
- **走廊 (hallways)**: 基础类型和接口
- **房间 (rooms)**: 高级概念（泛型、类、模块）
- **秘密通道 (secret-passages)**: 配置和工具
- **阁楼 (attic)**: AI辅助开发
- **地下室 (basement)**: 错误处理
- **鬼屋 (haunted-house)**: 项目开发阶段指南

## 技术栈

- **TypeScript**: 核心语言
- **Bun**: 运行时和包管理
- **React**: 前端组件（情绪切换器、维度叠加、幽灵助手）
- **XHTML/EPUB**: 内容格式
- **自定义构建系统**: 使用TypeScript编写的构建脚本

## 快速开始

```bash
# 安装依赖
bun install

# 开发模式（实时预览）
bun run dev

# 构建项目
bun run build

# 生成EPUB
bun run generate:epub

# 预览幽灵维度
bun run preview
```

访问 http://localhost:3000 进入幽灵维度。

## 幽灵维度功能

### 1. 情绪切换器 (EmotionSwitcher)
在幽灵维度的不同情绪状态间切换，影响界面色调和交互氛围。

### 2. 维度叠加 (DimensionOverlay)
可视化幽灵维度与常规维度的叠加，展示数据流动和边缘观察。

### 3. 幽灵助手 (GhostAssistant)
交互式幽灵助手，提供魔瓶视角的指导和观察。

### 4. 断面连接系统
所有断面通过目录系统相互连接，形成可穿梭的幽灵维度。

## 构建系统

项目包含完整的TypeScript构建系统：

- `scripts/build.ts`: 主构建脚本
- `scripts/watch.ts`: 开发监视脚本
- `scripts/generate-content.ts`: 内容生成器
- `scripts/generate-epub.ts`: EPUB生成器
- `scripts/preview.ts`: 预览服务器
